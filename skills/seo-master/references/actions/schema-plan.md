# /seo-master /schema-plan

## Purpose

Choose structured data that matches the page purpose and can be implemented without spam risk.

## Checklist

- Page type and eligible schema types
- Required and recommended properties
- Source of truth for each property
- JSON-LD placement
- Nesting opportunities
- Validation method
- Rich result eligibility
- Spam or mismatch risks

## Output

- Schema type recommendation
- Property map
- JSON-LD implementation notes or draft when requested
- Validation checklist
- Maintenance notes
